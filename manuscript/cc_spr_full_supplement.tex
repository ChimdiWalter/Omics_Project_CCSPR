\section{Important code-level correction}
The preprocessed single-cell loader originally reused the full stored PCA matrix whenever \texttt{X\_pca} was present in the \texttt{.h5ad} file, even if a smaller \texttt{n\_pcs} was requested in the configuration. This meant prior ``reduced'' configurations did not actually reduce the PCA feature dimension in the preprocessed path. The loader was corrected so the manuscript-safe run respects the requested PCA dimension.

\section{Manuscript-safe single-cell configuration}
The bounded single-cell policy is implemented by:
\begin{itemize}[leftmargin=1.5em]
\item \texttt{configs/cll\_rs\_scrna\_manuscript\_safe.yaml}
\item \texttt{experiments/run\_cll\_rs\_scrna\_manuscript\_safe.py}
\item \texttt{slurm/hellbender\_run\_cll\_scrna\_gse165087\_manuscript\_safe.sbatch}
\end{itemize}

The manuscript-safe configuration uses:
\begin{itemize}[leftmargin=1.5em]
\item \texttt{max\_cells = 320}
\item \texttt{n\_pcs = 20}
\item one train/test split
\item one harmonic bootstrap
\item one CC-SPR bootstrap
\item no ablation sweep
\end{itemize}

It also writes:
\begin{itemize}[leftmargin=1.5em]
\item dataset metadata JSON
\item checkpoint JSON
\item partial metrics and summary CSVs
\item figure-ready \texttt{.npz} arrays
\end{itemize}
These partial outputs are intended to prevent hours of silent execution and to preserve useful state if a bounded run is interrupted. In the completed manuscript-safe run, the logging and checkpoint path worked as intended and the run finished without approaching its memory limit.

\subsection{Resource-bounded single-cell ablations}
To add a minimal sensitivity analysis without recreating the earlier OOM / timeout pattern, the repository now includes:
\begin{itemize}[leftmargin=1.5em]
\item \texttt{configs/cll\_rs\_scrna\_ablation\_safe.yaml}
\item \texttt{experiments/run\_cll\_rs\_scrna\_ablation\_safe.py}
\item \texttt{scripts/run\_scrna\_ablation\_safe\_if\_needed.sh}
\item \texttt{slurm/hellbender\_run\_cll\_scrna\_gse165087\_ablation\_safe.sbatch}
\end{itemize}

This path is strictly smaller-risk than the previously failed scRNA sweeps and is intentionally limited to a single sensitivity axis:
\begin{itemize}[leftmargin=1.5em]
\item \texttt{max\_cells = 400}
\item \texttt{n\_hvg = 1000}
\item \texttt{n\_pcs = 25}
\item one train/test split
\item one harmonic bootstrap
\item one CC-SPR bootstrap
\item lambda-only sensitivity: $\lambda \in \{10^{-6}, 10^{-5}\}$
\item no multi-axis grid, no multi-split sweep, no multi-bootstrap sweep
\end{itemize}

The runner writes incremental state before the end of the job:
\begin{itemize}[leftmargin=1.5em]
\item checkpoint JSON
\item dataset metadata JSON
\item partial metrics CSV
\item partial main-results CSV
\item partial ablation-summary CSV
\item figure-ready \texttt{.npz} arrays
\end{itemize}
These files are the safeguard against silent login-node interruption and make the single-cell ablation path resumable in a way the earlier heavy runs were not.

\section{Implemented geometry-study scope}
The final paper is a geometry-sensitive sparse PH manuscript with empirical geometry claims restricted to the implemented Euclidean/Ricci study. Diffusion, PHATE-like, and DTM are retained in the theory and comparison framing because they are scientifically coherent backends for the method family, but they are not presented as completed benchmark results in this repository state.

Representative sparsity/entropy tables were also not recomputed post hoc. The cached outputs preserve TIP and H1 diagnostics, but not a full archive of per-bootstrap support vectors sufficient for honest entropy/support-size summaries without rerunning topology loops.

\section{What was intentionally deferred}
The following items were intentionally moved out of the critical path for the manuscript:
\begin{enumerate}[leftmargin=1.5em]
\item repeating the previously failed heavy single-cell configurations,
\item large single-cell ablation sweeps,
\item any multi-split or multi-bootstrap single-cell grid beyond the tiny bounded lambda check,
\item diffusion / PHATE-like / DTM backend experiments prior to implementation,
\item heavyweight survival or enrichment pipelines not already present in the repository outputs,
\item post hoc representative entropy/support-size recomputation that would require rerunning topology loops.
\end{enumerate}
